# Manifest Vision

## One-line vision

**Manifest is a calm, modern desktop workspace for structured projects, with first-class hierarchy, history, and meaningful change awareness.**

It should carry forward the best conceptual ideas from Archon — identity, snapshots, semantic change, and local-first reliability — while feeling like a **clean, intentional new product** rather than “Archon rewritten in another stack.”

---

## Positioning

Manifest is **not** intended to be a line-for-line port of Archon.

It is a greenfield product that should:

- feel more approachable and refined
- use a more mainstream desktop/web stack
- adopt a thinner, more explicit architecture
- earn each advanced feature on its own merits

---

## Product principles

### 1. Structure without heaviness
Users should be able to model complex projects and ideas in a hierarchy without feeling trapped in a rigid enterprise tool.

### 2. Change should be understandable
A core differentiator remains the ability to understand **what changed**, not just that a file changed.

### 3. Local-first and inspectable
Projects should be stored in formats that are durable, understandable, and recoverable.

### 4. Elegant by default
The default user experience should feel more like a thoughtful creative/productivity app than an engineering control panel.

### 5. Extensibility follows product clarity
Plugins and integrations matter, but only after the core experience is excellent.

---

## Likely target users

Manifest should initially optimize for people who need to manage **structured, evolving information**:

- product and research leads
- technical planners and architects
- operators with complex trees of assets, workflows, or systems
- solo builders and small teams who care about history and traceability

---

## Non-goals for v1

Manifest v1 should **not** try to be all of the following at once:

- a full clone of Archon
- a full real-time collaboration platform
- a giant integration marketplace
- a power-user CLI suite
- a complete plugin ecosystem from day one

---

## North-star experience

A user can:

1. create or open a structured project,
2. navigate and edit a hierarchy fluidly,
3. search instantly,
4. create named checkpoints/snapshots,
5. review meaningful changes over time,
6. trust that their work is recoverable and durable.

If Manifest gets those basics right with a polished desktop UX, it will already feel distinct and valuable.

---

## v1 success criteria

Manifest v1 is successful if it delivers:

- a polished hierarchy editor
- stable project storage and recovery
- fast local search
- named snapshots/history
- a meaningful diff experience
- a clean, tasteful desktop UI that feels lighter than Archon

---

## Product statement to keep in mind

> **Manifest should inherit Archon’s strongest ideas, not its obligations.**
